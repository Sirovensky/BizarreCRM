Design principles · research-agent consensus

Custom 64pt rail sidebar

SwiftUI has no native icon-only rail mode. Custom HStack of icon buttons + NavigationSplitView(columnVisibility: .detailOnly) suppresses the default sidebar. Rail width animates 64↔200pt on brand-mark tap. Active tab gets cream-tinted pill.

Inspector line-edit

.inspector(isPresented:) iOS 17+ with .inspectorColumnWidth(min:280, ideal:320, max:400). Applied to detail view, not root. Auto-fall-back to sheet on compact. Cart never hidden during edit — inspector pushes detail, not overlay.

Side-panel tender

Shopify POS v11 idiom. Items area swaps to tender UI; cart stays right, locked. Method grid + cash panel + quick-amounts + numpad, all inline scroll. Zero full-screen takeovers during a sale.

Cart collapses on paid

Post-sale: grid-template-columns: 64px 1fr 0. Animate 420→0px (240ms spring, 150ms opacity with Reduce Motion). Receipt + share tiles + actions fill the canvas. Next-sale CTA triggers cart re-expand.

Toolbar placements

Scan = .toolbar(.primaryAction), ⌘F1. Hold / Clear / Discount / Note / Misc = .secondaryAction — auto-collapse into trailing ⋯ menu on iPad. No bottom dock (Mac/Android pattern).

Smart Grid dynamic tiles

Catalog tiles respond to cart + customer state. Loyalty redeem tile when VIP customer attached. Ready-for-pickup shortcut surfaces her open ticket. Pattern from Shopify POS.

No drag-drop catalog→cart

No production POS ships drag-from-catalog-to-cart. Retail reasons: gloves, wet hands, trained staff prefer tap. Drag is reserved for layout editing only. Tap on tile is the primary gesture.

Keyboard shortcuts

Surfaced via Commands. ⌘N new sale · ⌘⇧N new ticket · ⌘K search · ⌘P pay · ⌘H hold · ⌘⇧C customer · ⌘⇧D discount. Shown inline in menu labels.

Pencil + Scribble

PKCanvasView for receipt signature + device photo annotation. Scribble free on every TextField (iPadOS 14+ automatic). drawingPolicy = .anyInput accepts finger + Pencil.

Hover + pointer

.hoverEffect(.highlight) on rail icons + toolbar buttons. .hoverEffect(.lift) on catalog tiles + customer card. .pointerStyle(.grab) only on layout-editor draggable tiles.

Multi-window

WindowGroup(id:for:) opens parked carts as separate Stage Manager windows. Shared @Observable model; per-scene @SceneStorage for UI state. Keyboard ⌘T = new window.

Compact fall-through

In Slide Over or 1/3 Split View (horizontalSizeClass == .compact) the iPad layout falls through to iPhone layout — rail hides, cart becomes a sheet. One SwiftUI view tree, two layouts.

Self-contained preview · no JS · no network · tokens mirror ios/Packages/DesignSystem/Sources/DesignSystem/BrandColors.swift + BrandFonts.swift. Ports directly to SwiftUI.

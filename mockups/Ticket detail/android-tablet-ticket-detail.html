Tablet ticket detail — mockup v2

  • NavigationRail at far-left, icon-only by default (80 dp wide). Active row highlighted in brand cream. Bottom of rail has an expand chevron — tapping switches to the labelled rail (200 dp) without leaving the screen.
  • Two-pane body 38 / 62 split, gated to sw >= 600. Phone keeps the existing single-column LazyColumn layout.
  • Bottom 5-icon bar removed. Tablet design language wants actions in the top app bar:
    • Status = primary CTA pill (cream) — most-tapped, deserves the dominant top-right slot.
    • Call / SMS / Note / Print = icon buttons, in that priority order.
    • Edit / Pin / ⋮ = secondary actions, after a divider.
  • Right pane compose bar gains a segmented selector — Note / SMS / Internal — so the same input field handles all three message kinds (most ticket-detail screens have separate sheets for each, which is friction). Status changes still go through the top-bar pill so they have permanent prominence.
  • Phone fallback (not shown): keep the bottom 5-action bar; right pane disappears; cards stack vertically. Same code path with a single BoxWithConstraints branch.

What changed from v1

  • NavigationRail (icon-only, 80 dp) restored at far-left so the screen size is gauge-able and the global nav stays reachable. Expand chevron at the bottom switches to the labelled 200 dp rail.
  • Bottom 5-icon action bar removed on tablet. All five actions promoted to the top app bar: Status as a cream pill (primary CTA), Call / SMS / Note / Print as icon buttons. Aligns with M3 tablet pattern (top-app-bar holds 3-5 visible actions, ⋮ for the rest).

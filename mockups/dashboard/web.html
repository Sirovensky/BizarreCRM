Design notes — Web

Decisions captured · 2026-05-04

Dynamic dashboard

  • Tile-based grid · 12-col · drag, drop, resize, toggle widgets
  • Library plan: react-grid-layout
  • Tile sizes 1×1 → 4×3 (web ceiling); each widget declares min/max
  • Customize button opens edit mode
  • Cross-device sync lives in Settings → Dashboard → Sync (not on toolbar)
  • Sync copies enabled-widget set across platforms; layout placement diverges per platform

Admin policy

  • Default: per-user layout
  • Admin can set: Per-user · Per-role · Store-locked
  • Resolver: user → role → store → default-preset
  • Defaults shipped per role: Owner / Tech / Cashier

Widget catalog (shared across platforms)

  • KPIs: Sales today, Open tickets, Avg ticket, Revenue MTD, Appts today
  • Greeting · Next appt · Unanswered messages · Needs attention
  • Sales week · Customer trends
  • Removed: Top repairs · Peak hour · "Store opens in 19 min"

Bell vs Comms

  • 🔔 Bell = system events only (parts arrived, payments, low-stock crossed)
  • 💬 Comms = unified customer inbox (SMS, calls, voicemail, email, chat)
  • Comms lives in sidebar with unread badge

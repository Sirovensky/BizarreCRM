Bizarre CRM POS - Canonical iOS Workflow

This file is now a canonical workflow reference. iOS keeps its own native expression: Liquid Glass chrome, solid readable content, safe-area CTAs on iPhone, and split-view production density on iPad. The workflow and server data stay identical to Android, tablet, and web POS.

Design system
Warm graphite + cream

Dark POS uses #FDEED0 for decisive action, active state, focus, and step progress. Surfaces remain neutral.

One server
Shared POS state

Customer, cart, repair draft, payment, receipt, and ticket records map to one backend contract across every platform.

iPhone identity
Thumb-first flow

Search lives in the navigation drawer, editors are sheets, and the primary tender action is always pinned in the safe area.

iPad identity
Persistent context

Catalog, cart, and repair/tender inspector stay side by side so staff can work quickly without losing the sale context.

1

Attach Customer

Search, scan loyalty, or continue as walk-in. Customer context is visible before cart decisions.

2

Add Work

Scan SKU, search service, add repair intake, or reserve parts from the same catalog source.

3

Edit Lines

Quantity, discount, device, warranty, condition, and notes use platform-native line editors.

4

Quote Repair

Issue, diagnostics, device condition, access, terms, deposit, and signature autosave as a draft.

5

Tender

Cash, card, split tender, store credit, deposit, balance due, and failure states share one payment intent.

6

Handoff

Print or send receipt, create ticket, print repair label, and return to a clean sale state.

Shared Server Objects

POSSession

Store, register, staff user, customer context, cart lifecycle, parked state, and audit trail.

CartLine

SKU, service, repair line, discount, warranty, tax, inventory reservation, and line notes.

RepairDraft

Issue, device, condition markers, tests, passcode policy, quote, ETA, terms, and signature.

PaymentIntent

Cash, card, split tender, store credit, deposit, balance due, failure recovery, and refunds.

Receipt

Printed receipt, SMS, email, tax details, customer copy, and repair drop-off receipt.

Ticket

Repair queue entry, thermal label, QR link, technician assignment, and pickup handoff.

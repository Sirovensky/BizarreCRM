IMPLEMENTATION
// androidx.compose.material3.SearchBar handles this transition natively.
// docked=false → full-screen expansion with built-in AnimatedVisibility.
var expanded by rememberSaveable { mutableStateOf(false) }
Box(Modifier.fillMaxSize()) {
    // Idle tiles layer — fades out when expanded
    AnimatedVisibility(
        visible = !expanded,
        enter  = fadeIn(spring(stiffness = StiffnessMediumLow)),
        exit   = fadeOut(spring(stiffness = StiffnessMediumLow))
    ) { EntryTiles(...) }

    SearchBar(
        query = query,
        onQueryChange = vm::onQueryChange,
        onSearch = vm::runSearch,
        active = expanded,
        onActiveChange = { expanded = it },
        placeholder = { Text("Customer, part, or ticket…") },
        leadingIcon = { Icon(Icons.Outlined.Search, null) },
        trailingIcon = {
            IconButton(onClick = vm::openScanner) {
                Icon(Icons.Outlined.PhotoCamera, "Scan")
            }
        },
        modifier = Modifier
            .fillMaxWidth()
            .align(if (expanded) Alignment.TopCenter else Alignment.BottomCenter)
            .padding(14.dp)
            .animateContentSize(spring(stiffness = StiffnessMediumLow))
    ) {
        // Results slot — renders when expanded=true
        SearchResults(query, onSelect = vm::attachCustomer)
    }
}
// Back handler collapses expanded state:
BackHandler(enabled = expanded) { expanded = false }

FLOW NOTES · v2b
  • Path branching — attached-customer screen shows 3 CTAs (Retail / Repair / Store credit). Most sales pick Retail; Repair adds Device picker + issue description steps. Store credit is inline top-up.
  • Device picker — shows devices already on file first, then "Add new" with IMEI scan (auto-fills make/model/year/warranty via lookup). Selected device attaches to both the cart AND the repair ticket draft.
  • Cart line tap — opens bottom sheet for qty/discount/note. Dim cart behind. Remove is destructive red, Save is primary. Swipe-left on line = quick remove.
  • Misc item · Note · Discount — three dashed-border slots under cart for power users. Misc = freeform one-off item (name + price). Note = prints on receipt. Discount = cart-wide %.
  • Tender split — applied tenders show top (store credit + cash); remaining balance dynamic. Add-tender grid stays accessible until balance = 0. Park cart = layaway mode.
  • Receipt send — SMS via BizarreSMS (default for mobile numbers), Email (default for email-only customers), Thermal print (always available if printer paired). Walk-in skips send, prints only.
  • Ticket link — if cart built from Repair path, "Open ticket #NNNN" CTA appears next to "New sale" so cashier can hand off to repair tech immediately.
  • Bottom search persists — on every screen except Tender + Receipt. Cashier can barcode-scan a new part mid-cart without losing context.

CHECK-IN FLOW NOTES
  • Autosave every step — ticket stays in draft state with autosave. Back button never loses data. Closing app → resume on next POS open (chip in customer header).
  • Skip-able steps — only Issue + Signature are required. Condition / Diagnostic / Access / Quote can be partial; tech can fill the rest post-drop-off from the desktop CRM.
  • Condition zone taps — device diagram is SVG; tap anywhere drops a marker with type picker (crack ✖ / scratch / dent / stain). Long-press = remove. Zoom via pinch.
  • Diagnostic tri-state — ✓/✕/? forces explicit "untested" so no silent assumptions. Mandatory for "cracked screen" tickets: touchscreen test before + after.
  • Passcode handling — stored in SQLCipher-encrypted column with TTL; nullified on ticket close. Audit log records read access by tech user id.
  • Quote is live — parts stock deducted on reserve (not on approve). Out-of-stock auto-switches to "Order" line with supplier ETA pulled from Mobilesentrix / PhoneLcdParts.
  • ETA calc — pulls latest supplier lead-time, adds tech queue depth (Mike has 2 ahead), adds labor minutes. Updates if supplier ETA slips.
  • Terms = shop-configurable — admin edits in Settings; versioned. Signed copy embeds the exact terms text + version hash for legal integrity.
  • Signature pad — captures pressure-curve SVG + timestamp + device meta. Stored alongside ticket. Tech's pickup-time signature uses same pad.
  • Print label — 2×1 inch thermal · ticket# / QR / customer last-name / device / drop-off date. QR opens ticket directly when scanned.

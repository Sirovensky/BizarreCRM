Parity check vs android-tablet-ticket-detail.html

  • Status pill — same cream + asymmetric corners. Tap = bottom sheet (Option B). ✓
  • Device card — same collapsed/expanded with chevron, same kv rows in expanded (First seen / Last seen gated on prior tickets > 0). ✓
  • Customer card — cookie-shape avatar, inline Call / SMS, same dedup rule (no duplicate buttons in top bar). ✓
  • Quote card — collapsed = total only; expanded = qrows + qtotals + Checkout. Add-line typeahead at the bottom with auto-classified PART / SVC / MISC suggestions. ✓
  • Photos — 4-up grid (same as tablet — fits 412 dp comfortably). ✓
  • Bench Timer — present, smaller clock font (22 vs 28). ✓
  • Activity feed — continuous rail, day dividers, oldest top → newest bottom, breathing pulse on Today. ✓
  • Compose bar — pinned bottom (vs tablet inline-with-timeline). Same Note / Diag / SMS segmented. Body padding 84 dp so last activity row stays visible above the bar. ✓
  • Print / Pin / Edit / Merge / Split / Sign-off / Delete — all under ⋮ overflow on phone (vs tablet's icon row + ⋮). ✓
  • Surface ramp — design-system Option F tokens (s-900 #0F0E0C → s-800 #161513 → s-700 #1E1C1A → #221F1B). ✓
  • Removed from tablet, also removed here: QR card (lives only on print label).
  • Differences from tablet by design: single column instead of 38/62 split; pill-row of inline icons (Print/Pin) collapses to ⋮ overflow; compose bar pinned to screen bottom rather than docked at the timeline pane bottom.
